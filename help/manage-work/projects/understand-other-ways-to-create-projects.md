# プロジェクトを作成するための4つの方法を検討する

ビデオでは、様々な方法を使用した Workfront でのプロジェクトの作成について包括的に説明します。

**テンプレートからのプロジェクトの作成：**

* プロジェクトページに移動し、「新規プロジェクト」をクリックして、「テンプレートから新規プロジェクト」を選択します。 &#x200B;
* アクティブなテンプレートを参照し、概要（期間、所有者、タスクの概要、カスタムフォームなど）をプレビューして、テンプレートを選択します。 &#x200B;
* お気に入りのテンプレートに直接アクセスして、すばやく選択できます。 &#x200B;
* 初期情報（プロジェクト名など）を入力し、「プロジェクトを作成」をクリックします。 &#x200B; フィールドは後で編集できます。 &#x200B;

**最初からのプロジェクトの作成：**

* プロジェクト ページに移動し、「新規プロジェクト」をクリックし、「新規プロジェクト」を選択します。 &#x200B;
* プロジェクトに名前を付け、共有オプション、カスタムデータ、ポートフォリオ、プログラムなどの詳細を編集します。 &#x200B;
* グループ管理者またはシステム管理者が設定したデフォルトは、承認または変更できます。 &#x200B;
* プロジェクトを保存します。初期段階では、作成者とシステム管理者のみが共有されます。 &#x200B;
* タスクは、名前を入力して追加し、「期間」や「予定時間」などのフィールドを編集できるようにします。 &#x200B;

**プロジェクトのコピー：**

* 既存のプロジェクトを開き、3点メニューをクリックして「コピー」を選択します。 &#x200B;
* 名前、ステータス（計画など）を変更し、コピーしない項目（費用など）の選択を解除します。 &#x200B;
* コピー後の予定開始日などの詳細を編集します。 &#x200B;

**Microsoft Project ファイルの読み込み：**

* MS プロジェクトファイルをXML ファイルとして保存します。 &#x200B;
* 新規プロジェクトメニューの「MS プロジェクトの読み込み」オプションを使用して、ファイルを選択し、読み込みを開始します。 &#x200B;
* 一部のMS プロジェクト設定（例：コメント、カスタムフォーム、マイルストーン）は転送されません。 &#x200B;
* ステータスを「計画」に設定し、読み込んだプロジェクトを公開前にレビューおよび編集します。 &#x200B;


このビデオでは、様々なユーザーのニーズに対応しながら、テンプレート、スクラッチ、コピー、読み込みを通じてプロジェクトを作成するWorkfrontの柔軟性を強調しています。 &#x200B;

## 重要な留意点

* **テンプレートによるプロジェクト作成の簡素化：**
* テンプレートを利用することで、タスク、フォーム、共有オプションなどの主要な詳細情報を事前に設定でき、時間と労力を節約できるため、利用を強くお勧めします。 &#x200B;
* **クイックアクセス用のお気に入りのテンプレート：**
* お気に入りのテンプレートを使用すると、ユーザーは選択プロセスをスキップしてプロジェクトを直接作成し、ワークフローを合理化できます。 &#x200B;
* **最初からのプロジェクトの作成による柔軟性の提供：**
* 手作業によるゼロからプロジェクトを作成する必要がある場合は、あらゆるディテールを詳細にカスタマイズできます。これは、単純なプロジェクトやプライベートなプロジェクトに最適です。 &#x200B;
* **プロジェクトのコピーによる時間の節約：**
* 既存のプロジェクトをコピーすると、その構造と設定が複製され、特定の要素を変更または除外するオプションが表示されます。 &#x200B;
* **移行用の MS Project ファイルの読み込み：**
* Workfrontでは、MS プロジェクトファイル（XMLとして保存）の読み込みがサポートされており、進行中のプロジェクトを移行できますが、一部のデータ（コメント、マイルストーンなど）は転送されません。 &#x200B;
