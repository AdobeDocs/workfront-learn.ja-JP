# タスク期間について

このビデオでは、Workfront でのタスク期間、タスクの開始日と完了日への影響、プロジェクトのタイムライン、期間を効果的に使用するベストプラクティスについて説明します。Workfrontでは、制約によって自動調整が妨げられることがあるため、手動で制約を設定するのではなく、期間と先行タスクを使用してタスクタイムラインを設定することをお勧めします。&#x200B;効果的なプロジェクト計画とリソース管理には、正確な期間が不可欠です。


>[!VIDEO](https://video.tv.adobe.com/v/3449344/?captions=jpn&quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **タスク期間の定義：** タスク期間は、タスクの予定開始日と予定完了日の差で、作業を完了するための時間枠を表します。 &#x200B;
* **プロジェクトのタイムラインに与える影響：** タスク期間は、プロジェクト全体の長さと予定完了日に直接影響するため、正確なプロジェクト計画に不可欠です。 &#x200B;
* **先行タスクの使用：**&#x200B;予定開始日と完了日は、タスク期間と先行タスクを使用して計算され、動的で柔軟なタイムラインを確保します。 &#x200B;
* **手動による制約を避ける：**&#x200B;開始日または完了日を手動で設定すると、タスクの制約が作成され、期間または先行タスクが変更されたときに自動調整が行われなくなる可能性があります。 &#x200B;
* **リソース管理：** Workfrontのリソース管理およびスケジューリングツールには、正確なリソース見積もりとチームの可用性プランニングを実現するために、期間が不可欠です。 &#x200B;
