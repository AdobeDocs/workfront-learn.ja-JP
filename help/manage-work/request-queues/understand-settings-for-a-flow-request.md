# リクエストフロー設定の整理

このビデオでは、キュー管理、ルーティングルール、トピックグループ、キュートピックに焦点を当てた、Workfrontでのリクエストフロープロセスについて説明します。 &#x200B;次の3つの重要な質問に答えることで、リクエストキューを整理する重要性を強調しています。

1. **どのトピックが必要ですか？** &#x200B; – 送信されたリクエストを分類するキューのトピックであり、管理やカスタマイズが簡単になります。 &#x200B;
1. **さらにトピックを整理できますか？** &#x200B; - トピックグループは、より良い整理のために関連するキューのトピックをグループ化するのに役立ちます（例：パンフレット、名刺、「プリント」の下の看板）。 &#x200B;
1. **リクエストは誰に割り当てるべきですか？** &#x200B; - ルーティングルールにより、送信が特定のチーム、役割、ユーザーに自動的に転送され、リクエストが見落とされないようにすることができます。 &#x200B;

これらの質問に答えることで、Workfrontでリクエストキューを効果的に計画および設定できます。 &#x200B;

>[!VIDEO](https://video.tv.adobe.com/v/3441906/?captions=jpn&quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **キューのトピック：**&#x200B;様々なタイプのリクエストを定義してキューのトピックに整理し、分類とカスタマイズを効率化します。 &#x200B;
* **トピックグループ：** トピックグループを使用して、キューのトピックをさらに整理し、関連するリクエストを簡単に見つけることができます（「印刷」および「デジタル」カテゴリのグループ化など）。 &#x200B;
* **ルーティングルール：** ルーティングルールを設定して、リクエストを特定のチーム、役割、またはユーザーに自動的に割り当て、送信の見落としを防ぎます。 &#x200B;
* **計画は必須です：** リクエストキューを構築する前に、必要な調整を行い、効率的な管理を実現します。 &#x200B;
* **通知されないリクエストを避ける：** ルーティングルールを使用しない場合、送信は「問題」タブに残り、手動で定期的にチェックしない限り通知されない場合があります。 &#x200B;
