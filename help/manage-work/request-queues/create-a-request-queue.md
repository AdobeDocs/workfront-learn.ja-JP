# Workfrontでのリクエストキューの作成

このビデオでは、Workfront内でのリクエストキューの作成に関するステップバイステップガイドを提供します。 &#x200B;このビデオでは、新しく作成されたキューを使用してリクエストを送信する方法を示し、詳細を入力し、「送信されたリクエスト」タブで送信を検証するプロセスを示します。 &#x200B;

>[!VIDEO](https://video.tv.adobe.com/v/335221/?quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **リクエストキューの作成：**&#x200B;まず、新しいプロジェクトを作成し、名前を付けます。これは後でリクエストキューとして機能します。 &#x200B; 「ヘルプリクエストキューとして公開」チェックボックスをオンにして、キューとして正式に指定します。 &#x200B;
* **アクセス権限：**&#x200B;誰、プロジェクトビューア、会社メンバー、グループメンバーの4つのオプションから選択して、リクエストをキューに送信できるユーザーを定義します。 &#x200B;
* **要求タイプ：** キューをカスタマイズして、問題、バグ報告、変更依頼、一般的な要求など、特定のタイプの要求を含めます。 &#x200B; デフォルトのオプションは「イシュー」です。 &#x200B;
* **カスタマイズ可能なフィールド：** デフォルトの応答時間、権限、その他のキュー特性などの追加の詳細を設定して、その機能をカスタマイズします。 &#x200B;
* **キューをライブ化：**&#x200B;設定を保存し、プロジェクトのステータスを「現在」に設定して、キューをアクティブにします。 &#x200B; リクエストを送信し、「リクエスト送信済み」タブで確認して、その機能をテストします。
