# Workfrontのリクエストキューの詳細

このビデオでは、Workfrontのリクエストキューのコンセプト、目的、およびリクエストの送信と管理方法について説明します。 リクエストキュー&#x200B;、さまざまなタイプのリクエストを送信するための一元的な場所として機能します。 

>[!VIDEO](https://video.tv.adobe.com/v/335220/?quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **リクエストキューの目的：** Workfront のリクエストキューでは、PTO、ソフトウェアアップグレード、クライアントの提案、トレーニングのニーズなど、様々なタイプのリクエストを送信する一元的な場所を提供します。
* **リクエストの送信：** ユーザーは、リクエスト領域に移動し、リクエストの種類を選択し、件名、説明、オプションのフィールド（ドキュメントなど）などのフィールドに入力し、カスタムフォームに入力することで、リクエストを送信できます。 &#x200B;
* **リクエストをイシューとして：** Workfrontでは、すべてのリクエストはイシューとして扱われ、プロジェクトに関連付ける必要があります。 リクエスト専用の&#x200B; プロジェクトは、リクエストキューと呼ばれます。 
* **キュー管理：**&#x200B;効果的なキュー管理には、要求をタイプ別に整理することが含まれます。これにより、必要な情報と割り当てがカスタマイズされ、効率が向上します。 &#x200B;
* **送信済みリクエストの検索：**&#x200B;送信済みリクエストは、関連するリクエストキューの「イシュー」タブで、「プロジェクト」エリアに移動して、関連するプロジェクトを選択することで見つけることができます。 &#x200B;
