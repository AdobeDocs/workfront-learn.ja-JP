# イシュー管理用のカスタムレポートの作成

このビデオでは、問題とプロジェクトを管理および追跡するための2種類のレポートを作成する手順を説明します。 &#x200B;

* **問題レポート：**&#x200B;このレポートは、問題が割り当てられたか、タスクまたはプロジェクトに変換されたか（オブジェクトを解決中）を追跡するのに役立ちます。 &#x200B; オブジェクトの解決やエントリ日などの列が含まれ、最新のエントリで並べ替えられます。 フィルターは、まだ完了していない特定のリクエストキューの問題のみを表示するために適用されます。 
* **プロジェクトレポート：**&#x200B;このレポートは、問題から発生したプロジェクトに焦点を当て、元の問題とプロジェクトの進捗状況に関する詳細を表示します。 &#x200B;変換済みのイシューのエントリ日、イシュー名、発信者名、プロジェクトのステータス、進行状況などの列が含まれます。 フィルターを使用すると、イシューからコンバージョンされたプロジェクトのみが表示され、まだ完了していません。 

この動画では、特定のニーズに合わせてレポートをカスタマイズするための列とフィルターのカスタマイズに重点を置いており、問題の効率的な管理と解決が容易になっています。 &#x200B;

## 重要な留意点

* **問題管理のためのカスタムレポート：**&#x200B;このビデオでは、未解決の問題を一元管理して追跡するためのカスタムレポートを作成する方法を説明し、問題の管理と解決を容易にします。 &#x200B;
* **問題レポートの詳細：**&#x200B;問題レポートには、解決オブジェクトやエントリ日などの列が含まれており、最新のエントリで並べ替えられています。 &#x200B; イシューが割り当てられたか、タスクやプロジェクトに変換されたかを特定するのに役立ちます。 
* **プロジェクトレポートの詳細：**&#x200B;プロジェクトレポートは、イシューから発生したプロジェクトに焦点を当て、変換済みのイシューのエントリ日、イシュー名、発信者名、プロジェクトのステータス、進捗ステータスなどの詳細が表示されます。
* **関連性に関するフィルタリング：**&#x200B;両方のレポートでフィルターを使用して、関連データのみを表示します。 問題レポートの場合、フィルターには、完了していない特定のリクエストキューの問題が表示されます。 &#x200B; プロジェクトレポートの場合、フィルターを使用すると、イシューから変換されたプロジェクトのみが表示され、まだ完了していません。 
* **カスタマイズと効率：**&#x200B;ビデオでは、列とフィルターのカスタマイズの重要性をハイライト表示して、特定のニーズに合わせてレポートを調整し、イシューとプロジェクトの管理の効率を高めます。
