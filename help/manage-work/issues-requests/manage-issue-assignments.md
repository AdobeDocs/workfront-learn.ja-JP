# 問題の割り当てを効果的に管理

イシューの管理は、プロジェクトマネージャーや、イシューに関する作業に割り当てられたユーザーの重要な責務です。

このビデオでは、以下の方法を説明します。

* イシューの割り当て
* イシュー割り当てを検索
* イシューの進捗の更新
* イシューステータスの更新

>[!VIDEO](https://video.tv.adobe.com/v/3446956/?captions=jpn&quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **問題の割り当て：**&#x200B;割り当てフィールドのインライン編集を使用して、ユーザーまたはチームに問題を割り当て、割り当て後に「保存」をクリックしてください。 &#x200B; チームリードは、ワークロードバランサーを使用して、チームメンバーのカレンダーにタスクをドラッグアンドドロップすることで割り当てることができます。 &#x200B;
* **割り当てられた問題の検索：**&#x200B;割り当てられた問題がホーム作業リストに表示され、フィルタリングできます。 チ&#x200B;ムに割り当てられたイシューは、チームリクエスト セクションに表示され、チームメンバーはタスクを承認または再割り当てできます。 &#x200B;これらの問題は、見つけやすくするためにワークロードバランサーにも表示されます。 &#x200B;
* **進行状況の更新：**&#x200B;更新後の更新時間と問題発生時のログ時間を記録して、作業の進行状況を可視化します。 &#x200B; 「名前」機能を使用して、特定のユーザーまたはチームにコメントを送信します。 &#x200B;通知はコメント用に送信されますが、ログ時間やステータスの変更時には送信されません。 &#x200B;
* **問題の状態を変更：**&#x200B;問題の状態を更新して、「作業中」や「完了」などの進行状況を反映します。 未解決の問題&#x200B;は、関連するタスクやプロジェクトが完了としてマークされるのを防ぎます。 &#x200B;
* **チーム固有のワークフロー：**&#x200B;組織の固有のワークフローとシステム設定に従って、問題を効果的に管理します。 &#x200B;
