# 予定外の作業を効果的に処理する

この動画では、プロジェクト管理におけるコミュニケーションの重要性を強調し、コミュニケーションを促進する重要なツールとしての課題の役割を強調しています。 &#x200B;プロジェクトを完了する前に割り当て、追跡、解決する必要がある作業に適している、更新とイシューの違いを説明しています。 &#x200B;このビデオでは、問題をログに記録する方法をステップバイステップガイドで説明します。 


>[!VIDEO](https://video.tv.adobe.com/v/3419488/?quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **イシューとアップデート：** イシューは、プロジェクトの完了前に割り当て、追跡、解決が必要なタスクに使用します。イシューは、アップデートと比較して監視および整理が容易です。 &#x200B;
* **問題のログ記録：**&#x200B;問題は、「問題」タブを使用して、タスクまたはプロジェクトレベルでログに記録できます。 &#x200B;優先度を設定し、割り当てや更新に応じてメール通知を設定できます。
* **問題管理のベストプラクティス：**&#x200B;問題を頻繁に監視し、効率を高めるためにカスタムレポートを作成し、未解決の問題について明確な説明を提供して、繰り返し質問を回避し、履歴レコードを作成します。 &#x200B;
* **コミュニケーションの促進：** チームメンバーに問題を記録し、効果的なコミュニケーションを促し、円滑なプロジェクト管理を実現するため、回答に感謝の意を示します。 &#x200B;
* **問題をタスクに変換：**&#x200B;問題をタスクに変換し、プロジェクトのタイムラインに統合することで、より優れた計画と実行を実現できます。 &#x200B;
