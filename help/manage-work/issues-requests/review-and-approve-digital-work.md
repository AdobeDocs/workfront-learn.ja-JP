# デジタル作品のレビューと承認

このビデオでは、ホームエリアの「承認待ち」ウィジェットを使用して、デジタル作業をレビューおよび承認する方法について説明します。 &#x200B;次の5種類の承認について説明します。

* **プロジェクトの承認：** コメントを追加するオプションを使用して、ホーム領域で直接承認または却下できます。 &#x200B;詳細なレビューを行うには、プロジェクト名をクリックして詳細にアクセスします。 &#x200B; プロジェクト内で行われた承認では、コメントの入力を求めるプロンプトは表示されませんが、オプションでコメントを入力できます。
* **タスクの承認：**&#x200B;プロジェクトの承認と同様に、タスクの名前をクリックしてタスクを確認したり、コメントを付けて直接承認／却下したりすることができます。
* **イシューの承認：**&#x200B;タスク／プロジェクトの承認と同様に機能します。コメントを追加せずに承認をスキップするオプションが用意されています。
* **プルーフの承認：** プルーフに移動してレビューし、特定のコメントを追加します。 &#x200B;承認オプションには、「承認」、「変更で承認」、「変更が必要」、「関連性なし」などがあります。
* **ドキュメントの承認：** プルーフのないドキュメントは、引き続き承認または拒否できます。 &#x200B;変更が必要な場合は、プルーフの承認で「変更を加えて承認」と同様に、コメントに変更を記述します。

この動画では、作業のレビューと承認の柔軟性が強調されており、必要に応じてコメントを追加したり、変更を指定したりできるオプションが用意されています。 &#x200B;

## 重要な留意点

* **一元的な承認管理：**&#x200B;承認待ちのすべてのアイテムは、ホーム領域の「承認待ち」ウィジェットでアクセスできます。 &#x200B;
* **柔軟なレビューオプション：** ホーム領域から直接承認または却下するか、項目名をクリックして詳細をレビューしてから決定できます。 &#x200B;
* **注釈オプション：** コメントの追加はオプションですが、明確にすることをお勧めします。 &#x200B; ホーム領域で行われた承認はコメントを求めますが、アイテム内で行われた承認はコメントを求めません。 
* **プルーフの承認詳細：** プルーフの承認には、「変更を承認済み」、「変更が必要」、「関連性がありません」など、複数のオプションがあり、詳細なフィードバックが可能です。 &#x200B;
* **変更の処理：**&#x200B;変更が必要な文書またはプルーフの場合、コメントの変更を指定して、再提出または更新のための明確なコミュニケーションを確保できます。 &#x200B;


>[!NOTE]
>
>承認が必要なドキュメント、プルーフおよび作業項目（プロジェクト、タスク、イシュー）は、[!DNL Workfront Home] だけでなくプロジェクトやカスタムレポートにもあります。
