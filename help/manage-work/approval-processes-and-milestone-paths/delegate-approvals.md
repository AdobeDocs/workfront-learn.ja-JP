# タスク、イシュー、承認の効果的な委任

このビデオでは、タスク、イシュー、承認を委任する方法について説明します。&#x200B;委任を有効にするには、システム管理者またはグループ管理者がプロジェクト環境設定の「設定」で機能を有効にする必要があります。&#x200B;有効にすると、ユーザーはマイタスクウィジェットの「デリゲート」ボタンを使用して、すべてのタスクとイシューを指定した期間だけ別のユーザーに委任できます。&#x200B;委任は、タスクとイシューの両方に適用され、同じユーザーに割り当てる必要があります。&#x200B;委任は、終了日の有無にかかわらず設定でき、ユーザーはいつでも手動で委任を停止でき&#x200B;す。
承認の場合、ユーザーは「承認待ち」ウィジェットを使用して承認を委任できます。&#x200B;委任は、指定された期間内の既存の承認と新しい承認に適用されますが、プロジェクト、タスク、イシューに対してのみ適用されます。ドキュメント、プルーフ、タイムシートには適用されません。&#x200B;委任されたタスク、イシュー、および承認の通知は、管理者がデリゲータとデリゲートの両方に通知できるように有効にすること&#x200B;できます。
委任された承認はシステム内でマークされ、誰が誰に委任したかを示します。&#x200B;承認に関連する更新は「更新」タブに表示され、システムアクティビティは「システムアクティビティ」タブに表示でき&#x200B;す。

## 重要な留意点

* **委任の有効化：**&#x200B;タスク、イシュー、承認の委任は、まずシステム管理者またはグループ管理者がプロジェクト環境設定の設定で有効にする必要があります。
* **タスクとイシューのデリゲート：** ユーザーは、マイタスクウィジェットの「デリゲート」ボタンを使用して、すべてのタスクとイシューを（個別ではなく）特定の期間に1人のユーザーに委任できます。 &#x200B;委任は、終了日の有無にかかわらず設定でき、手動で停止できます。
* **承認の委任：** プロジェクト、タスク、および問題の承認は、承認待ちウィジェットの「承認を委任」ボタンを使用して委任できます。 &#x200B;ただし、ドキュメント、プルーフ、タイムシートの承認は委任できません。
* **通知：**&#x200B;管理者は、委任されたタスク、イシュー、承認に関するメール通知を管理者が有効にして、委任を割り当てた際や停止した際に委任者と委任先の両方に通知できます。
* **委任のトラッキング：**&#x200B;委任されたタスクと承認は、システム内でマークされ、誰が委任し、誰に委任したかを示します。 &#x200B;承認に関連する更新は「更新」タブに表示され、システムアクティビティは「システムアクティビティ」タブに表示されます。
