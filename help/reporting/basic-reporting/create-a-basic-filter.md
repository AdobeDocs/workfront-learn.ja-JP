# 基本フィルターの作成

このビデオでは、Workfrontでフィルターを作成して使用し、特定の条件に基づいてリストレポートをカスタマイズする方法を説明します。 &#x200B; フィルターを使用すると、条件を定義することで、タスク、プロジェクト、ドキュメントなどのリストを絞り込むことができます。 &#x200B; フィルター条件に一致し、ログインユーザーがアクセス可能な項目のみがリストに表示されます。 &#x200B;


>[!VIDEO](https://video.tv.adobe.com/v/335151/?quality=12&learn=on&enablevpops=1)

## 重要な留意点

* **フィルター一覧レポートのカスタマイズ：** Workfrontのフィルターを使用すると、一覧（タスク、プロジェクト、ドキュメントなど）を絞り込むことができます 特定の条件にもとづいて、条件に一致し、ログインユーザーがアクセスできるアイテムのみを表示します。 &#x200B;
* **一般的なフィルターの種類：**&#x200B;例には、「マイプロジェクト」（ログインしたユーザーに固有のプロジェクトを表示）などのユーザーベースのフィルターと、「今月の期限」（今月の完了予定日を含むアイテムを表示）などの日付ベースのフィルターがあります。 &#x200B;
* **フィルターの作成：** フィルターは、「AND」フィルター（すべての条件がtrueである必要があります）や「OR」フィルター（任意の条件がtrueである可能性があります）などのルールを使用して作成されます。 &#x200B;例えば、「自分が完了したタスク」フィルターは、ログインユーザーに割り当てられた完了したタスクとタスクのルールを組み合わせます。 &#x200B;
* **フィルターの共有と削除：** フィルターは、他のWorkfront ユーザーと共有したり、フィルターメニューを介して削除したりできるため、柔軟性が高く、共同作業に適したツールになります。 &#x200B;
* **明確な名前付け方法：** フィルター名に「My」または「I」を使用すると、フィルターの結果が自分に固有であることをユーザーが理解しやすくなり、わかりやすさと使いやすさが向上します。 &#x200B;
